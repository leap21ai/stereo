Under the Hood
Five modules that make markdown executable.
Each one handles a different part of the pipeline — from parsing your .md to rendering live output inline.
Markdown Parsing
PARSER
Extracts ```lang run blocks from standard markdown with zero config.
Live Output
RENDERER
Renders live output inline with OKLCH-themed dashboard components.
Isolation
SANDBOX
Executes code in a restricted VM with no access to fs or process.
HTTP Proxy
SIDECAR
Proxies HTTP and injects secrets so credentials never touch JS.
UI Library
COMPONENTS
Ships MetricCard, Sparkline, StatusTable, and DashGrid built-in.
How It Works
Three steps. No build system.
Standard markdown in, live dashboards out. Nothing to configure.
1
Write Markdown
Create a standard .md file. Annotate code blocks with ```tsx run. That's the only syntax Stereo adds.
2
Open Preview
Press Ctrl+Shift+M. Stereo renders your markdown with live output areas alongside each executable block.
3
Run Blocks
Click Run or set auto-refresh. See dashboards, API data, and charts rendered inline — no terminal needed.
Auto-refresh built in. Set refresh=30s on any block. Data stays current without manual reruns.

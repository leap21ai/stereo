API Health Dashboard

Last updated: Mar 12, 2026 · Auto-refreshes every 30s · Trey Anderson

Real-time health status for production API services. This runbook auto-queries the /health endpoint and renders current status inline.

live tsx · auto-refresh 30s
const res = await fetch("{{env.API_URL}}/health");
const data = await res.json();
 
<DashGrid cols={3}>

    <MetricCard
      label={s.name}
      value={s.latency + "ms"}
      status={s.healthy ? "success" : "danger"}
    />

</DashGrid>

Stereo Split Markdown
01

```tsx run

The magic annotation. Any fenced code block with run becomes executable. Add refresh=30s for auto-polling. The block renders its output inline, replacing the code fence. Click "Source" to toggle back.

02

{{env.API_URL}}

Mustache syntax for credential injection. The Rust proxy sidecar reads your .env file and injects values at the network layer. JavaScript never sees the raw secret. Works with any HTTP call inside a run block.

03

Built-in Components

<DashGrid>, <MetricCard>, <Sparkline>, <StatusTable> — a small standard library of dashboard primitives. Styled with Tailwind. Import your own from node_modules.

04

Three View Modes

Rendered: WYSIWYG with live output inline. Source: raw markdown with syntax-highlighted run blocks. Split: source on left, live output on right. Toggle via command palette or status bar.

05

AI Writes Markdown

Claude Code, Copilot, Cursor — they all edit .md files natively. The extension watches for changes and hot-reloads run blocks. No special integration needed. The agent writes a code fence, the extension runs it.

06

Status Bar: Stereo

Green dot = proxy running, blocks executing. Shows count of live blocks. Click to open the Stereo panel: proxy logs, env var status, block execution history, error traces.
